# Primus TouchDesigner Components (`.tox`)

Phase 9 creates live Base COMPs inside a `.toe`. Export them to this folder so
future shows can drag them from a Palette.

## Export steps (inside TouchDesigner)

1. Run Phase 9 builder in TouchDesigner:

```python
exec(open(f'{project.folder}/builders/phase9_components.py').read())
build()
```

2. For each component under `/project1/primus_phase9/`:
   - `PrimusManager`
   - `device_template` (save as `PrimusDevice.tox`)
   - `PrimusCueEngine`

3. Right-click the COMP → **Save Component…** → save into this `tox/` directory:

| COMP | Suggested filename |
|------|--------------------|
| `device_template` | `PrimusDevice.tox` |
| `PrimusManager` | `PrimusManager.tox` |
| `PrimusCueEngine` | `PrimusCueEngine.tox` |

4. Optional Palette: Preferences → Palette → add folder pointing at `tdprimus/tox`.

## Fresh-project smoke test

1. New empty `.toe`
2. Drop `PrimusManager.tox` + `PrimusCueEngine.tox`
3. Pulse **Rescan** → devices table fills
4. Pulse **Rebuild** → Device comps appear
5. Set IPs if needed; Identify pulse on a Device
6. Cue Engine **GO** advances cues

## Note

`.tox` binaries are not generated by this repo’s scripts — only created from TD. Keep them out of git if they become large; the builder scripts are the source of truth.
